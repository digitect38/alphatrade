# KOSPI/KOSDAQ 장 운영시간 및 휴장일 가이드

AlphaTrade 시스템이 참조하는 한국 주식시장 운영 스케줄.

---

## 1. KOSPI/KOSDAQ 세션 스케줄 (KST)

| 세션 | 시간 (KST) | 매매 방식 | 설명 |
|------|-----------|----------|------|
| 장전 시간외 (Pre-market) | 08:00 - 08:30 | 단일가 | 전일 종가 기준 +/-10% 범위 |
| 동시호가 (Opening auction) | 08:30 - 09:00 | 단일가 | 개장가 결정, 호가 접수만 가능 |
| 정규장 (Regular session) | 09:00 - 15:20 | 접속매매 (연속) | 일반 매매 시간 |
| 장마감 동시호가 (Closing auction) | 15:20 - 15:30 | 단일가 | 종가 결정 |
| 장후 시간외 (After-hours) | 15:40 - 16:00 | 단일가 | 당일 종가 기준 매매 |

### 참고사항
- 반일장 (설/추석 전일 등): 정규장 09:00-13:20, 장마감 동시호가 13:20-13:30
- 동시호가 시간에는 체결 없이 호가만 접수되며, 종료 시점에 단일가로 일괄 체결
- 장전/장후 시간외 매매는 종가 기준 단일가이므로 가격 변동 없음

## 3. 시스템 내 장 운영시간 활용

### 3.1 세션 가드 (TradingGuard)

신규 진입 허용 시간을 정규장 내로 제한한다.

- **허용 구간**: 09:05 ~ 15:00 (장 개시 후 5분, 장 마감 전 20분)
- **차단 구간**: 동시호가, 장전/장후 시간외, 휴장일, 정규장 외 시간
- **구현**: `execution/risk_manager.py` 내 `TradingGuard.check_session()`
- **설정**: `config.py`의 `ENTRY_START_OFFSET_MIN=5`, `ENTRY_END_OFFSET_MIN=20`

### 3.2 스캐너 타이밍 (Scanner Timing)

| 스캐너 | 실행 시점 | 근거 |
|--------|----------|------|
| Morning Scanner | 08:50 - 09:05 | 동시호가 데이터 기반 갭/거래량 급증 탐색 |
| Intraday Scanner | 09:05 이후 5분 간격 | 정규장 실시간 모멘텀/거래량 스캔 |
| EOD Scanner | 15:35 - 16:00 | 장 마감 후 당일 실적 집계 |

### 3.3 n8n 워크플로우 스케줄

| 워크플로우 | Cron 표현식 | 설명 |
|-----------|------------|------|
| WF-01 데이터 수집 | `0 8 * * 1-5` | 장 시작 1시간 전 전일 데이터 수집 |
| WF-02 모닝 스캔 | `50 8 * * 1-5` | 동시호가 직전 스캐너 트리거 |
| WF-03 트레이딩 루프 | `*/5 9-14 * * 1-5` | 정규장 중 5분 간격 실행 |
| WF-04 EOD 조정 | `35 15 * * 1-5` | 장 마감 후 포지션/잔고 대조 |
| WF-05 일간 리포트 | `0 16 * * 1-5` | 장후 시간외 종료 후 일간 성과 리포트 |

모든 n8n 워크플로우는 평일(월-금)에만 실행되며, 공휴일 체크는 워크플로우 첫 노드에서 `GET /market/session` API를 호출하여 `closed`인 경우 즉시 종료한다.

### 3.4 KIS WebSocket 연결 관리

- **연결 시작**: 08:25 (장전 시간외 매매 5분 전)
- **구독 종목 갱신**: 09:00 (모닝 스캐너 결과 반영)
- **연결 종료**: 16:05 (장후 시간외 종료 후)
- **휴장일**: 연결 시도하지 않음

## 4. API 레퍼런스: GET /market/session

현재 시장 세션 상태를 조회한다.

### 엔드포인트

```
GET /market/session
```

### 응답 예시

```json
{
  "session": "regular",
  "market": "KOSPI/KOSDAQ",
  "current_time": "2026-03-30T10:30:00+09:00",
  "session_start": "09:00",
  "session_end": "15:20",
  "is_trading_day": true,
  "entry_allowed": true,
  "next_session": {
    "name": "closing_auction",
    "starts_at": "15:20"
  }
}
```

### 응답 필드

| 필드 | 타입 | 설명 |
|------|------|------|
| `session` | string | 현재 세션. `pre_market`, `opening_auction`, `regular`, `closing_auction`, `after_hours`, `closed` 중 하나 |
| `market` | string | 시장 구분 (`KOSPI/KOSDAQ`) |
| `current_time` | string (ISO 8601) | 서버 현재 시각 (KST) |
| `session_start` | string | 현재 세션 시작 시각 (HH:MM) |
| `session_end` | string | 현재 세션 종료 시각 (HH:MM) |
| `is_trading_day` | boolean | 오늘이 거래일인지 여부 (휴장일/주말이면 `false`) |
| `entry_allowed` | boolean | 현재 신규 진입 허용 여부 (세션 가드 기준) |
| `next_session` | object/null | 다음 세션 정보. 당일 마지막 세션이면 `null` |

### 세션 값 상세

| 값 | 시간 | 설명 |
|----|------|------|
| `pre_market` | 08:00 - 08:30 | 장전 시간외 |
| `opening_auction` | 08:30 - 09:00 | 동시호가 |
| `regular` | 09:00 - 15:20 | 정규장 |
| `closing_auction` | 15:20 - 15:30 | 장마감 동시호가 |
| `after_hours` | 15:40 - 16:00 | 장후 시간외 |
| `closed` | 그 외 | 장 마감 / 휴장 |

### 사용 예시

```bash
# 현재 세션 확인
curl http://localhost:8000/market/session

# n8n 워크플로우에서 휴장일 체크
# → session이 "closed"이고 is_trading_day가 false이면 워크플로우 중단
```
